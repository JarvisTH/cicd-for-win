💡 填写路径后自动检测项目类型、代码检查规则和 Git 远程仓库
选择流水线操作的目标分支，留空则使用当前检出分支
▶ 🚀 部署配置
部署 = 将构建产物通过 SFTP 上传到远程服务器并启动服务。
💡 没有密钥？只需配置一次：
1️⃣ 在本机运行 ssh-keygen -t ed25519（一路回车即可）
2️⃣ 把公钥（.pub 文件）添加到服务器的 ~/.ssh/authorized_keys
3️⃣ 这里填私钥路径：C:\Users\你的用户名\.ssh\id_ed25519
▶ 📤 Git 远程仓库
同时推送到多个远程仓库，如 GitHub + Gitee。
▶ 🔍 代码检查规则
代码检查规则 在流水线 check 步骤执行时自动运行。
规则按项目类型自动匹配，可单独启用/禁用。规则文件位于 CI/CD 目录的 rules/ 下，独立于项目源码，不入侵项目本身。
如需自定义规则，直接修改对应规则文件即可。
▶ ⚙️ 流水线配置
自定义流水线步骤的启用/禁用和执行顺序。拖拽步骤可调整先后顺序。
